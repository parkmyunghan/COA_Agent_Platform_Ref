# 지휘통제 및 분석 페이지 UX 개선 계획

## 현재 문제점 분석

1. **중복 제목**: Header와 페이지 내부에 동일한 제목이 중복 표시
2. **시스템 상태 카드**: 4개 카드가 상단에 고정되어 공간을 많이 차지
3. **축선별 전장분석 요약**: 전체 너비를 사용하지 않아 공간 낭비
4. **정황보고 배너**: 상세 내용이 많아 공간을 많이 차지

## UX 개선 방안

### 1. 중복 제목 제거 ✅
- **조치**: 페이지 내부 제목 제거, Header의 제목만 유지
- **효과**: 수직 공간 절약, 정보 계층 구조 명확화

### 2. 시스템 상태 카드 처리
- **제안 A (권장)**: 핵심 정보만 컴팩트하게 표시
  - 시스템 엔진 상태만 Header에 작은 인디케이터로 표시
  - 나머지 정보는 필요시에만 접근 (툴팁, 드롭다운)
- **제안 B**: 다른 페이지로 이동
  - 데이터 관리 센터나 별도 상태 페이지로 이동
  - 하지만 실시간 위협 수는 이 페이지에서도 유용할 수 있음

**최종 결정**: 제안 A 채택 - Header에 시스템 상태 인디케이터 추가, 상세 정보는 필요시 접근

### 3. 축선별 전장분석 요약 개선
- **현재**: 전체 너비를 사용하지 않음
- **개선안**: 
  - 좌측 패널 상단으로 이동하여 공간 효율성 향상
  - 또는 지도 하단 상세 정보 패널과 통합
  - 또는 좌측 패널 내부에 탭/섹션으로 통합

**최종 결정**: 좌측 패널 상단에 컴팩트하게 배치

### 4. 정황보고 간소화
- **현재**: 
  - SituationBanner가 별도 섹션으로 표시 (상세 내용 포함)
  - 지도 내부에도 정황보고 표시 (중복)
- **개선안**:
  - SituationBanner 제거 또는 최소화
  - 지도 내부 정황보고만 유지 (이미 구현됨)
  - 필요시 지도 내 정황보고를 클릭하면 상세 정보 표시

**최종 결정**: SituationBanner 제거, 지도 내부 정황보고만 유지

### 5. 추가 개선 사항
- **좌측 패널 최적화**: 
  - Agent 선택, 시스템 설정, 상황 입력을 더 컴팩트하게 배치
  - 접을 수 있는 섹션으로 구성
- **지도 하단 상세 정보 패널**:
  - 축선별 전장분석 요약과 통합 고려
  - 탭 구조로 전환 가능

## 구현 우선순위

1. **High Priority**:
   - 중복 제목 제거
   - 정황보고 배너 제거/간소화
   - 시스템 상태 카드 최소화

2. **Medium Priority**:
   - 축선별 전장분석 요약 위치 최적화
   - 좌측 패널 레이아웃 개선

3. **Low Priority**:
   - 추가 UX 개선 (애니메이션, 인터랙션 등)
