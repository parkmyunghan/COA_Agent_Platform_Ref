# 차세대 방책 추천 시스템 발전 로드맵 (COA System Roadmap)

본 문서는 현재의 '라이브러리 기반 방책 추천 시스템'을 진단하고, Palantir AIP 등 선도적인 플랫폼과 비교하여 향후 시스템이 나아가야 할 3단계 발전 방향을 제시합니다.

---

## 1. 현황 진단 및 비교 (Current Status vs Palantir AIP)

### 1.1 현재 우리 시스템 (Library Selection Model)
현재 시스템은 **"검증된 교리(Template)의 선택"**에 초점을 맞추고 있습니다.

*   **방식**: 사전에 정의된 `COA_Library`(완성형 방책 템플릿) 데이터베이스에서 현재 상황(위협, 환경)에 가장 적합한 방책을 검색(Retrieval)하고 순위를 매깁니다.
*   **장점**: 군사적으로 검증된, 신뢰성 높은 방책만을 제시하므로 안정적이고 설명 가능합니다(Explainable).
*   **한계**: 데이터베이스에 미리 정의되지 않은 변칙적이거나 복합적인 상황에 대해서는 유연한 대응이 어렵습니다.

### 1.2 Palantir AIP 모델 (Generative Orchestration Model)
Palantir의 AIP는 **"단위 기능의 조합(Orchestration)"**을 지향합니다.

*   **방식**: 완성된 방책이 아니라 '작전의 최소 단위(Block)'를 라이브러리로 보유하고, AI가 이를 실시간으로 조립하여 상황에 맞는 새로운 방책을 생성(Generation)합니다.
*   **특징**: "타격", "정찰", "기동" 등의 기능을 상황에 맞춰 레고 블록처럼 결합합니다.

---

## 2. 단계별 발전 로드맵 (Roadmap)

시스템의 유연성과 창의성을 극대화하기 위해 다음 3단계 진화 모델을 제안합니다.

### [Phase 1] 파라메트릭 최적화 (Parametric Adaptation) - *단기 목표*
기존 라이브러리 방책을 유지하되, 세부 속성(Parameter)을 AI가 상황에 맞춰 미세 조정하는 단계입니다.

*   **개념**: Template Selection + Parameter Tuning
*   **주요 기능**:
    *   **위치 최적화**: "후방 지역 방호" 방책을 선택하되, 적의 예상 진격 속도를 고려하여 *"주 방어선을 템플릿보다 5km 후방에 설치"*하도록 제안.
    *   **자원 할당 조정**: 기상 악화 시 항공 자산 대신 포병 할당량을 자동으로 20% 증량.
*   **기술 요구사항**: LLM이 텍스트 생성을 넘어, JSON 형태의 작전 매개변수(좌표, 수량, 시간)를 수정하도록 프롬프트 엔지니어링 고도화.

### [Phase 2] 모듈형 방책 생성 (Modular COA Construction) - *중기 목표*
온톨로지 구조를 '방책 단위'에서 '과업(Task) 단위'로 해체하여 재조립하는 단계입니다.

*   **개념**: Task Retrieval + Dynamic Orchestration
*   **주요 기능**:
    *   **모듈 단위 검색**: "산악 지형 정찰(Task A)", "대규모 화력 타격(Task B)", "공중 강습(Task C)" 모듈을 각각 검색.
    *   **동적 조립**: AI가 위 3가지 모듈을 결합하여 *"산악 거점 확보를 위한 공중 강습 및 화력 지원 작전"*이라는, 라이브러리에 없던 새로운 방책을 생성.
*   **기술 요구사항**:
    *   온톨로지 스키마 재설계 (COA 중심 → Capability/Task 중심)
    *   Task 간의 상호 호환성(Interoperability) 및 제약 조건(Constraint) 검증 로직 구현.

### [Phase 3] 시뮬레이션 검증 루프 (Wargaming Feedback Loop) - *장기 목표*
생성된 방책의 승률을 예측 데이터(통계)가 아닌 실시간 시뮬레이션으로 검증하는 단계입니다.

*   **개념**: AI Generation + Wargame Validation
*   **주요 기능**:
    *   **자동 모의 분석**: AI가 생성한 100가지 변형 방책을 경량 워게임 엔진에서 시뮬레이션 수행.
    *   **최적해 도출**: "예비대를 좌측방이 아닌 우측방에 배치했을 때, 아군 생존율이 15% 상승함"과 같은 정량적/확률적 근거 제시.
*   **기술 요구사항**:
    *   고속 경량 시뮬레이터(Lightweight Wargame Engine) 연동.
    *   강화학습(RL) 기반의 자율 에이전트 도입.

---

## 3. 결론 (Conclusion)

현재의 라이브러리 기반 시스템은 **'신뢰성(Reliability)'** 측면에서 강력한 기반입니다. 향후 경쟁력을 확보하기 위해서는 **Phase 1(최적화)**을 시작으로, 점진적으로 **Phase 2(생성)** 및 **Phase 3(검증)**로 나아가, AI가 단순한 정보 검색 도구가 아닌 **'창의적이고 검증된 작전 참모'**의 역할을 수행할 수 있도록 발전시켜야 합니다.
